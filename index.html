AI Integration at RDB

The KPI programme is built around a single thesis: AI becomes useful when it’s integrated into the tools we already use every day. Sheets, ClickUp, report pipelines, SOPs. Every section below is a concrete integration — a place where an LLM plugs into real work and removes a manual step.

What “AI integration” means here

Not a chatbot bolted onto the side of a workflow. Real integration means the AI lives inside the tool — a custom function in a spreadsheet cell, a skill Claude runs against a folder of reference files, a prompt library your team edits like any other doc.

Everything in the Exercises track is a working example: you’ll end up with an AI-powered Google Sheet, a reusable data-cleaning skill, and three tuned system prompts your team actually ships with.

Exercise: Create a data-cleaning.md Using Claude

In this exercise you will use the free Claude website (claude.ai) to generate a reusable data-cleaning.md instruction file. You will prompt Claude to write the file for you, then download and save it.

💡Requirement: A free Claude account at claude.ai. No paid plan needed.
📋

Objective

Generate a Markdown file called data-cleaning.md that serves as a reusable instruction set for cleaning media monitoring datasets. The file should cover:

  • Removing duplicate rows based on URL or headline
  • Filtering out rows containing unwanted keywords
  • Enriching date columns (extract Month, Day, Year)
  • Outputting a cleaned file ready for analysis
🛠

Step-by-Step Instructions

1

Go to the Claude website

Open claude.ai and log in with your free account (or create one).

2

Start a new conversation

Click "New chat" to begin a fresh conversation.

3

Enter the following prompt

📋 Prompt to Copy
Create a Markdown file called data-cleaning.md that serves as a
reusable instruction guide for cleaning media monitoring CSV/Excel
datasets. The file should include:

1. **Purpose** — Explain that this is for cleaning raw media
   monitoring data

2. **Input Requirements** — Describe expected columns:
   Date, Headline, URL, Publication, Content

3. **Cleaning Steps:**
   - Duplicate Removal: deduplicate by URL (fallback to Headline)
   - Keyword Filtering: remove rows with user-specified keywords
     (case-insensitive)
   - Date Enrichment: parse the Date column and add Month, Day,
     Year columns

4. **Output** — Save as a new file with _cleaned suffix,
   include a summary of changes

5. **Rules** — Never modify the original file, preserve
   column order, handle missing Date column gracefully

Format it as a clean, well-structured Markdown document that
someone could follow as step-by-step instructions. Output the
full file content inside a code block.
4

Review Claude's output

Claude will generate the full Markdown. Review it and follow up, e.g.:

  • "Add a section for handling encoding issues in CSV files"
  • "Include an example of the expected output summary format"
  • "Make the keyword filtering section more detailed"
5

Save the file

  1. Click the Copy button on the code block
  2. Open any text editor (Notepad, VS Code, etc.)
  3. Paste and save as data-cleaning.md
6

Test it

Start a new Claude chat, upload a sample CSV, and paste data-cleaning.md as instructions. Verify that Claude follows the cleaning steps correctly.

✨

Expected Output (Reference)

Your generated data-cleaning.md should look similar to this:

📄 data-cleaning.md
# Data Cleaning Instructions

## Purpose
Clean raw media monitoring datasets by removing duplicates,
filtering unwanted keyword rows, and enriching date columns.

## Input Requirements
- File format: CSV or Excel (.xlsx)
- Expected columns: Date, Headline, URL, Publication, Content
- Additional columns are preserved as-is

## Cleaning Steps

### 1. Duplicate Removal
- Primary key: **URL** column
- Fallback: **Headline** column (if URL is missing)
- Keep the first occurrence; remove subsequent duplicates
- Report: "Removed X duplicate rows"

### 2. Keyword Filtering
- Ask the user for a list of keywords to exclude
- Check Headline and Content columns (case-insensitive)
- Remove any row that contains an excluded keyword
- Report: "Filtered X rows matching excluded keywords"

### 3. Date Enrichment
- Auto-detect the date column (Date, Published, Pub Date)
- Parse into YYYY-MM-DD format
- Add new columns at the end: Month, Day, Year
- Supported formats: MM/DD/YYYY, DD-Mon-YYYY, ISO 8601

## Output
- Save as: [original_filename]_cleaned.[ext]
- Print summary: original rows, duplicates removed,
  rows filtered, final row count

## Rules
- Never modify the original file
- Preserve all original columns in their original order
- New columns (Month, Day, Year) go at the end
- If no Date column exists, skip enrichment and notify user
💬

Tips for Better Results

  • Be specific — the more detail you give Claude, the better the output
  • Iterate — ask Claude to revise sections that don't match your needs
  • Add examples — ask Claude to include sample input/output for each step
  • Test with real data — upload an actual CSV in a follow-up chat to verify the instructions work
  • Version your file — save improved versions as data-cleaning-v2.md, etc.

const DS_CONFIG1 = {
    DAILY_REQUEST_LIMIT: 500,
    API_ENDPOINT: 'https://api.deepseek.com/v1/chat/completions',
    MODEL: 'deepseek-chat',
    MAX_TOKENS: 4096,
    TEMPERATURE: 0.0,
    REQUEST_RETRIES: 2,
    REQUEST_TIMEOUT_MS: 30000,
    PROMPT_DOC_IDS: {
        sentiment: '1JP6WecOtrxfHRbCZK6ROacEFRQaIjfPGENfBUb8IM1M',
        summary:   '1kXMZSSrPNx0coqpzqtkzVmGdCfpzJlbUpzfQfR9Hp7M',
        category:  '15c5Xu3wEerypXUGgfclGAodAZg8pJdhyieN1TS9iS5Q'
    }
};

function setupCredentials(apiKey) {
    const key = (apiKey || '').toString().trim();
    if (!key) throw new Error('Please provide an API key. Example: setupCredentials("sk-...")');
    PropertiesService.getScriptProperties().setProperty('DS_API_KEY', key);
    return 'API key saved successfully.';
}

function setApiKeyFromPrompt() {
    const ui = SpreadsheetApp.getUi();
    const response = ui.prompt('Set DeepSeek API Key',
        'Paste your API key (it will be saved in Script Properties):',
        ui.ButtonSet.OK_CANCEL);
    if (response.getSelectedButton() !== ui.Button.OK) { ui.alert('Cancelled.'); return; }
    ui.alert(setupCredentials(response.getResponseText()));
}

function clearApiKey() {
    PropertiesService.getScriptProperties().deleteProperty('DS_API_KEY');
    return 'API key cleared.';
}

function getApiKey_() {
    const apiKey = PropertiesService.getScriptProperties().getProperty('DS_API_KEY');
    if (!apiKey || !apiKey.trim())
        return { ok: false, error: 'Error: API key is missing. Use Prompt Tools > Set API Key.' };
    return { ok: true, value: apiKey.trim() };
}

function syncPromptFromDoc(taskName) {
    try {
        const cleanTaskName = normalizeTaskName(taskName || 'sentiment');
        const scriptProps = PropertiesService.getScriptProperties();
        const configuredDocId = (DS_CONFIG1.PROMPT_DOC_IDS && DS_CONFIG1.PROMPT_DOC_IDS[cleanTaskName]) || '';
        const propertyDocId = scriptProps.getProperty(getPromptDocIdKey(cleanTaskName));
        const docId = configuredDocId || propertyDocId;
        if (!docId) return 'Error: Prompt Doc is not configured for "' + cleanTaskName + '".';
        const docText = DocumentApp.openById(docId).getBody().getText().trim();
        if (!docText) return 'Error: Prompt Doc for "' + cleanTaskName + '" is empty.';
        scriptProps.setProperty(getPromptTextKey(cleanTaskName), docText);
        return 'Prompt synced successfully for "' + cleanTaskName + '".';
    } catch (error) {
        return 'Error: Unable to read Prompt Doc - ' + error.toString();
    }
}

function syncAllPromptsFromDocs() {
    return ['sentiment','summary','category'].map(syncPromptFromDoc).join('\n');
}

function getCachedSystemPrompt(taskName) {
    const cleanTaskName = normalizeTaskName(taskName || 'sentiment');
    const promptText = PropertiesService.getScriptProperties()
        .getProperty(getPromptTextKey(cleanTaskName));
    if (!promptText || promptText.trim() === '')
        return 'Error: Prompt for "' + cleanTaskName + '" is not synced yet.';
    return promptText;
}

function normalizeTaskName(taskName) {
    const valid = ['sentiment','summary','category'];
    const n = String(taskName || '').trim().toLowerCase();
    if (valid.indexOf(n) === -1) throw new Error('Invalid taskName. Use: sentiment, summary, category.');
    return n;
}
function getPromptTextKey(n){return 'PROMPT_TEXT_' + n.toUpperCase();}
function getPromptDocIdKey(n){return 'PROMPT_DOC_ID_' + n.toUpperCase();}

function parseJsonSafe_(text){ try{return JSON.parse(text);}catch(e){return null;} }

function callDeepSeek_(payload, apiKey) {
    const options = {
        method: 'post',
        contentType: 'application/json',
        headers: { 'Authorization': 'Bearer ' + apiKey, 'Content-Type': 'application/json' },
        payload: JSON.stringify(payload),
        muteHttpExceptions: true,
        timeout: DS_CONFIG1.REQUEST_TIMEOUT_MS
    };
    let attempt = 0;
    const maxAttempts = DS_CONFIG1.REQUEST_RETRIES + 1;
    while (attempt < maxAttempts) {
        attempt++;
        try {
            const response = UrlFetchApp.fetch(DS_CONFIG1.API_ENDPOINT, options);
            const status = response.getResponseCode();
            const bodyJson = parseJsonSafe_(response.getContentText() || '');
            if (status === 200) return { ok: true, data: bodyJson || {} };
            const retryable = status === 429 || status >= 500;
            if (retryable && attempt < maxAttempts) { Utilities.sleep(1000 * attempt); continue; }
            const msg = (bodyJson && bodyJson.error && bodyJson.error.message) || ('API failed with status ' + status);
            return { ok: false, error: 'Error: ' + msg, status: status };
        } catch (error) {
            if (attempt < maxAttempts) { Utilities.sleep(1000 * attempt); continue; }
            return { ok: false, error: 'Error: Network/API request failed - ' + error.toString() };
        }
    }
    return { ok: false, error: 'Error: API request failed after retries.' };
}

function checkRateLimit() {
    const cache = CacheService.getScriptCache();
    const today = new Date().toDateString();
    const requestCount = cache.get(today);
    if (!requestCount) { cache.put(today, '1', 24*60*60); return true; }
    const count = parseInt(requestCount);
    if (count >= DS_CONFIG1.DAILY_REQUEST_LIMIT) return false;
    cache.put(today, (count + 1).toString(), 24*60*60);
    return true;
}

function runAiTask(taskName, text_to_analyze) {
    if (!text_to_analyze || text_to_analyze.toString().trim() === '') return "Cell is empty";
    if (!checkRateLimit()) return 'Error: Daily request limit (' + DS_CONFIG1.DAILY_REQUEST_LIMIT + ') exceeded.';
    const apiKeyResult = getApiKey_();
    if (!apiKeyResult.ok) return apiKeyResult.error;
    const systemPrompt = getCachedSystemPrompt(taskName);
    if (systemPrompt.indexOf('Error:') === 0) return systemPrompt;
    const payload = {
        model: DS_CONFIG1.MODEL,
        messages: [
            { role: 'system', content: systemPrompt },
            { role: 'user',   content: text_to_analyze }
        ],
        max_tokens: DS_CONFIG1.MAX_TOKENS,
        temperature: DS_CONFIG1.TEMPERATURE
    };
    const result = callDeepSeek_(payload, apiKeyResult.value);
    if (!result.ok) return result.error;
    const content = result.data && result.data.choices && result.data.choices[0]
        && result.data.choices[0].message && result.data.choices[0].message.content;
    if (!content) return 'Error: API returned an unexpected response format.';
    return String(content).trim();
}

/** @customfunction */ function sentimentTagger(text){ return runAiTask('sentiment', text); }
/** @customfunction */ function summarizer(text){     return runAiTask('summary',   text); }
/** @customfunction */ function categoryTagger(text){ return runAiTask('category',  text); }

❄ FreezeFunc.gs
const AI_CUSTOM_FUNCTIONS = ['sentimentTagger','summarizer','categoryTagger'];
const AI_FORMULA_REGEX = new RegExp('=\\s*(' + AI_CUSTOM_FUNCTIONS.join('|') + ')\\s*\\(','i');

/** Freeze AI formulas in the ACTIVE sheet only. */
function freezeApiFormulasActiveSheet() {
  const sheet = SpreadsheetApp.getActiveSheet();
  const range = sheet.getDataRange();
  const formulas = range.getFormulas();
  const values   = range.getValues();
  let frozen = 0;

  for (let r = 0; r < formulas.length; r++) {
    for (let c = 0; c < formulas[r].length; c++) {
      const f = formulas[r][c];
      if (f && AI_FORMULA_REGEX.test(f)) frozen++;
      else if (f) values[r][c] = f;
    }
  }
  if (frozen > 0) range.setValues(values);
  SpreadsheetApp.getActive().toast(
    frozen + ' AI formula(s) frozen in "' + sheet.getName() + '"',
    'Freeze AI Formulas', 5);
}

Prompt 1 — Summary

Reference Template

📄 Summary Prompt — Google Doc
ROLE:
You are a senior media analyst at a PR monitoring agency.
You specialize in extracting key information from news
articles and producing concise, executive-ready summaries
for client reports.

INSTRUCTION:
Summarize the provided news article text. Your summary must:
- Capture the main story in 1-2 sentences
- Identify WHO is involved (people, organizations)
- State WHAT happened or was announced
- Note WHY it matters or its significance
- Include any relevant numbers, dates, or data points

Output a single paragraph summary of 40-60 words.
Do not use bullet points in the output.

CONSTRAINTS:
- Maximum 60 words; never exceed this limit
- Write in third person, past tense
- Do not include your own opinions or analysis
- Do not add information not found in the original text
- If the text is too short or unclear, respond with:
  "Insufficient content for summary"
- Do not start with "The article discusses..." or
  "This article is about..."
- Use neutral, factual language throughout

SAMPLE OUTPUT:

Input: "Tech giant Acme Corp announced a $50M expansion
of its Manila operations on March 10, creating 2,000 new
jobs in AI development. CEO Maria Santos said the move
positions the Philippines as a key hub for the company's
Asia-Pacific growth strategy."

Output: "Acme Corp announced a $50 million expansion of
its Manila operations on March 10, projected to generate
2,000 new AI development positions. CEO Maria Santos
emphasized the investment strengthens the Philippines'
role in the company's Asia-Pacific growth strategy."

Prompt 2 — Sentiment Tagging

Powers =sentimentTagger(). Classifies article text by sentiment toward your client.

1

Create a new Google Doc named "Sentiment Prompt"

2

Adjust labels to match your agency's reporting standards.

Reference Template

📄 Sentiment Prompt — Google Doc
ROLE:
You are a media sentiment analyst specializing in brand
reputation monitoring. You assess news coverage to determine
how it reflects on the subject organization, using
standardized sentiment categories for PR reporting.

INSTRUCTION:
Analyze the sentiment of the provided news article text
toward the main subject (person, company, or organization
mentioned most prominently). Classify it into exactly ONE
of the following categories:

- POSITIVE: Coverage that favorably portrays the subject
  (achievements, awards, growth, praise, endorsements,
  successful initiatives)
- NEGATIVE: Coverage that unfavorably portrays the subject
  (scandals, failures, criticism, losses, controversies,
  legal issues)
- NEUTRAL: Factual reporting with no clear positive or
  negative framing (announcements, routine coverage,
  industry data, event listings)
- BALANCED: Coverage that contains BOTH clearly positive
  AND clearly negative elements in roughly equal measure

Output ONLY the sentiment label. Nothing else.

CONSTRAINTS:
- Respond with exactly ONE word: POSITIVE, NEGATIVE,
  NEUTRAL, or BALANCED
- Do not include explanations, justifications, or analysis
- Do not use quotation marks in your response
- Judge sentiment toward the PRIMARY subject
- Press releases and routine announcements are NEUTRAL
- Opinion pieces evaluated by stance, not topic
- If text is empty or not a news article, respond: NEUTRAL

SAMPLE OUTPUT:

Input: "Acme Corp reported record-breaking Q1 revenue of
$2.3 billion, exceeding analyst expectations by 15%."
Output: POSITIVE

Input: "Acme Corp announced it will hold its annual
shareholders meeting on April 15."
Output: NEUTRAL

Input: "Acme Corp faces a class-action lawsuit from
former employees alleging wage theft."
Output: NEGATIVE

Input: "Acme Corp's new sustainability initiative earned
praise, but also drew criticism for delaying emissions
targets by two years."
Output: BALANCED

Reference Template

📄 Category Prompt — Google Doc
ROLE:
You are a media content classifier working for a PR
monitoring agency. You categorize news articles into
predefined topic themes to support media analysis
reporting and share-of-voice tracking.

INSTRUCTION:
Read the provided news article text and assign it to
the single MOST relevant category from the list below:

CATEGORIES:
- Corporate News (mergers, acquisitions, leadership)
- Financial (earnings, revenue, stock, investments)
- Product/Service (launches, reviews, updates)
- CSR/Sustainability (environmental, ESG)
- Legal/Regulatory (lawsuits, compliance)
- Industry/Market (trends, analysis, forecasts)
- Technology/Innovation (R&D, patents, AI)
- People/HR (hiring, layoffs, culture)
- Crisis/Issues (scandals, PR crises)
- Events (conferences, trade shows, summits)

Output ONLY the category name. Nothing else.

CONSTRAINTS:
- Choose exactly ONE category (the most dominant theme)
- Use the exact category name from the list
- Do not create new categories or combine them
- If article spans themes, choose the one with majority content
- If empty or not a news article, respond: Corporate News
- Do not include explanations or reasoning

SAMPLE OUTPUT:

Input: "Acme Corp announced acquisition of Beta Inc
for $300 million, expanding in Southeast Asia."
Output: Corporate News

Input: "Acme Corp launched its new AI-powered analytics
platform at CES 2026."
Output: Product/Service

Input: "Acme Corp's CEO spoke at WEF about the pledge
to net-zero emissions by 2030."
Output: CSR/Sustainability

Input: "The Philippine SEC opened an investigation into
Acme Corp's financial disclosures."
Output: Legal/Regulatory

Slide 07 / 12FreezeFunc

Freeze AI Formulas

Every spreadsheet recalculation re-runs AI functions, consuming tokens. FreezeFunc solves that.

What it does

  • Scans the active sheet for AI-function formulas
  • Replaces them with their evaluated text values
  • Preserves all other formulas untouched
  • Single batched write for efficiency

Prompt Tools > Output > Freeze AI Formulas (Active Sheet)

⚠Irreversible: once frozen, original formulas are replaced.

Slide 09 / 12Rate Limiting & Security

Rate Limiting

  • Soft daily limit of 500 requests (configurable)
  • Counter stored in CacheService with 24-hour TTL
  • Shared across all users of the deployment

Security Notes

  • API key stored in Script Properties (per-script isolation)

  • Custom functions run with the opening user’s permissions

What is Prompt & Context Engineering?

📝 Prompt Engineering

Crafting precise instructions for AI to produce the desired output. The clearer your ask, the better the result.

📂 Context Engineering

Providing the right reference materials, background info, constraints so the AI understands the task domain.

💡Think of it this way: A prompt is the question. Context is everything the AI needs to answer it well.

Why it matters in media analytics: report accuracy, categorization consistency, and reproducible workflows all depend on how well you communicate with the AI.

🔑Mindset shift: treat Claude as a collaborator, not just an executor.

Slide 05 / 12Reverse Engineering

Reverse Engineering Reports into SOPs

Instead of writing an SOP from scratch, start with the finished report and work backwards. Reports are the most complete, real-world documentation of how work actually gets done.

1

Collect existing client deliverables — reports, dashboards, scoring sheets

2

Identify the structure — sections, data fields, categorization logic

3

Extract the implicit process — what steps did the analyst follow?

4

Document decision points — where does human judgment apply vs. fixed rules?

5

Feed reference files into the SOP Generator to auto-create the interactive SOP

✓Why this works: reports capture real analyst behaviour — the single best source of ground truth for building SOPs.
Slide 06 / 12Best Practices

Prompt Engineering for SOP Generation

🎯 Be Specific About Context

Include client name, industry, platforms monitored, reporting cadence. The more the AI knows, the less it guesses.

📎 Include All Reference Files

The more complete the input, the fewer clarifying questions the skill asks.

🏗 Use Structured Instructions

"Analyze the attached files and generate an SOP covering categorization rules, exclusion criteria, QA steps."

🔄 Iterate on Outputs

Review first output, then refine: "Add a QA checklist for Section 3" / "Expand exclusion rules".

💡Tell the AI what to prioritise: categorisation rules, exclusion criteria, QA steps, formatting standards.
